# OrgAsm Manual

OrgAsm v0.29 - двухпроходный ассемблер Z80 для компьютера Sprinter под DSS. Исторический автор - Igor Zhadinets. Текущая версия сопровождается Dmitry Mikhalchenkov.

## Требования

Для запуска `ORGASM.EXE` нужна целевая среда Sprinter с BIOS 3.xx и DSS Estex.

## Командная строка

```text
OrgAsm [drv:\path\]inFile[.ext] [drv:\path\outFile.ext] [/options]
```

Если у входного файла нет расширения, добавляется `.asm`. Если выходной файл не указан, он создается рядом с исходником с расширением `.exe`.

Опции:

- `/E` или `-E` - создать EXE-префикс. Адрес загрузки берется из первой `ORG`, адрес запуска из `ENTRY`, стек из `STACK`.
- `/C` или `-C` - считать регистр букв значимым в именах меток.
- `/L`, `/L:name.err`, `/L=name.err` - создать файл ошибок только при наличии ошибок.
- `/M` - создать таблицу символов.
- `/N` - не создавать неявный выходной файл; выход создается только директивами `SAVE`/`SAVEBIN` и явно запрошенными служебными файлами.
- `/S` - очистить экран перед работой.

## Диагностика

Ошибки компиляции выводятся на экран и не создают выходной файл. С ключом `/L` ошибки записываются в общий файл для основного исходника и всех `INCLUDE`-файлов:

```text
file.asm:12: Syntax error
    исходная строка
```

Файл `.ERR` создается только после первой ошибки. OrgAsm выходит через DSS `EXIT #41` с кодом возврата `0` в регистре `B` после успешной сборки и `1`, если ошибки компилятора или DSS остановили создание выходных файлов. Фатальные ошибки DSS выводятся текстовым сообщением DSS, например `Invalid filename` или `No free memory`.

Во время компиляции OrgAsm опрашивает клавиатуру между строками исходника и во время длинных операций загрузки/сохранения файлов. `Ctrl+C` отменяет текущую сборку: OrgAsm выводит сообщение об отмене, закрывает открытые файлы, освобождает выделенную память и выходит с кодом возврата `1`.

## Формат строки

```asm
Label           Instruction ; comment
```

Метка должна начинаться в первой колонке. Инструкция не должна начинаться в первой колонке. Несколько инструкций можно писать в одной строке через `:`, кроме строк с `INCLUDE` и `INCBIN`:

```asm
Loop:           ld (hl),a: inc hl: djnz Loop
```

Имена меток могут содержать латинские буквы, цифры и `_`, но не могут начинаться с цифры.

## Локальные метки

Локальная метка начинается с точки и видима до следующей обычной метки:

```asm
Test1:          ld b,16
.loop:          inc hl
                djnz .loop

Test2:          ld b,32
.loop:          inc de
                djnz .loop
```

К локальной метке можно обратиться полным именем, например `Test1.loop`.

## Числа и выражения

Поддерживаются десятичные, шестнадцатеричные и двоичные числа:

```asm
#a000
0C000h
0x8F12
%00110011
%0100'0000
10010110b
0b01010101
```

В двоичных числах можно использовать апостроф как разделитель групп.

Символ `$` означает адрес текущей инструкции. Символы в одинарных или двойных кавычках дают код символа.

Поддерживаемые операции:

- унарные: `!`, `^`, `-`, `+`, `++`, `--`, `<`, `>`, `?Label`;
- сдвиги: `<<`, `>>`;
- арифметика: `*`, `/`, `%`, `\`, `+`, `-`;
- сравнения: `<`, `=`, `>`, `<=`, `!=`, `<>`, `>=`;
- битовые операции: `!`, `&`, `|`.

Результаты выражений 16-битные; при переполнении используются младшие 16 бит.

## Поддерживаемые директивы

### ORG

```asm
                org #8100
```

Задает адрес размещения кода. Если `ORG` отсутствует, используется `#8100`.

### ENTRY

```asm
                entry Start
```

Задает адрес запуска для EXE-префикса. При отсутствии `ENTRY` используется адрес загрузки.

### STACK

```asm
                stack #bfff
```

Задает SP для EXE-префикса. Значение по умолчанию - `#bfff`.

### PHASE / DEPHASE / DISP / ENT

```asm
                phase #4000
Relocated:      ret
                dephase

                disp #7000
Virtual:        byte #01
                ent
```

Меняют адрес компиляции без изменения адреса записи объектного кода. `DISP` является TASM-совместимым алиасом `PHASE`, а `ENT` - алиасом `DEPHASE`. Вложенный `PHASE`/`DISP` и `ORG` внутри блока запрещены.

### EQU

```asm
PChars          equ #5c
```

Присваивает метке постоянное значение. Выражение не должно ссылаться вперед.

### DB / DEFB / BYTE

```asm
Text:           db "Hello",13,10,0
                byte #3e,#2a
```

Генерирует байты и строки.

### DW / DEFW / WORD

```asm
Table:          dw Start,Exit
                word #4000
```

Генерирует 16-битные слова в little-endian.

### DD / DEFD / DWORD

```asm
Longs:          dd #5678
Ptr32:          defd Start
HeaderValue:    dword #1234
```

Генерирует 32-битные значения в little-endian. `DWORD` является TASM-совместимым алиасом. Выражения вычисляются как 16-битные значения и дополняются нулевым старшим словом.

### DUP / EDUP

```asm
                dup 3
                byte #11
                edup
```

Повторяет строки между `DUP` и `EDUP` заданное число раз. Поддерживается один активный блок `DUP`; вложенные блоки не поддерживаются.

### DS / DEFS / BLOCK

```asm
Buffer:         ds 256
Spaces:         ds 10," "
Padding:        block 16,#ff
```

Резервирует и заполняет область памяти. Без второго параметра используется ноль.

### INCLUDE

```asm
                include "inc/text.asm"
```

Подключает исходный файл. Если расширение отсутствует, добавляется `.asm`. Вложенные `INCLUDE` поддерживаются, максимум 64 исходных файла за одну сборку.

### INCBIN

```asm
Sprite:         incbin "sprite.bin"
Part:           incbin "data.bin",16,32
```

Вставляет бинарный файл целиком или диапазон по смещению и длине.

### SAVE / SAVEBIN

```asm
                org #4000
Start:          byte #3e,#2a
End:
                savebin "CODE.BIN",#4000,End-#4000
                save "COPY.BIN",#4000,End-#4000
```

Сохраняют диапазон скомпилированного кода в бинарный файл после успешного второго прохода. Запись откладывается до конца компиляции, поэтому при ошибках частичные файлы не создаются. Если запрошенная длина выходит за пределы реально сгенерированного объектного кода, сохраняется только доступная часть диапазона. Для сборки только через явные `SAVE`/`SAVEBIN` используйте ключ `/N`.

### OUTPUT / OUTEND

```asm
                org #4000
                output "RANGE.BIN"
Start:          byte #3e,#2a
                outend
                savebin "COPY.BIN",#4000,$-#4000
```

`OUTPUT` начинает отложенную запись диапазона с текущего адреса компиляции, `OUTEND` завершает диапазон на текущем адресе. Имя файла должно быть указано в кавычках. Вложенные диапазоны `OUTPUT` не поддерживаются.

За одну сборку можно задать до 8 отложенных выходных файлов через `SAVE`, `SAVEBIN` и `OUTPUT`; имя выходного файла в директиве ограничено 63 символами.

### Диагностика

```asm
                display "Building DIAG example"
                warning "Check build flags"
                assert 1,"Diagnostic check"
                error "Unsupported build option"
```

`DISPLAY` и `WARNING` печатают сообщение во время сборки. `ERROR` печатает сообщение и завершает сборку диагностикой `User error`. `ASSERT` вычисляет выражение и при нулевом результате завершает сборку диагностикой `Assertion failed`; после запятой можно указать необязательное сообщение в кавычках. Сообщения `DISPLAY`, `WARNING` и `ASSERT` выводятся на втором проходе, поэтому не дублируются; активный `ERROR` завершает сборку сразу. Строки сообщений должны быть заключены в одинарные или двойные кавычки.

### Условная компиляция

```asm
                define BUILD 1

                ifdef BUILD
Start:          byte #3e,#42
                else
Skipped bad source line
                endif

                ifn 0
                byte #32
                word Value
                endif

                if 0
Broken:         db ?
                elseif BUILD
Value:          byte #24
                endif

                ifndef NEWFLAG : define NEWFLAG 7 : endif
                ifdef NEWFLAG : byte NEWFLAG : endif
```

`DEFINE name value` добавляет числовое имя, которое затем можно использовать в выражениях и проверках `IFDEF`; `UNDEFINE name` удаляет ранее заданное имя. Поддерживаются `IF expr`, `IFN expr`, `IFDEF name`, `IFNDEF name`, `ELSEIF expr`, `ELSE` и `ENDIF`. Выражения условий должны быть вычислимы в точке директивы. Неактивные ветки пропускаются до конца строки без разбора меток, команд и некорректного кода. Условные директивы можно объединять с другими директивами через `:`, например `IFNDEF X : DEFINE X 1 : ENDIF`.

## Особенности Z80-синтаксиса

Поддерживаются документированные команды Z80 и некоторые распространенные расширения:

- `in a,(bc)` и `out (bc),e` принимаются как варианты `(c)`;
- `(ix)` и `(iy)` принимаются как `(ix+0)` и `(iy+0)`;
- `EX AF,AF'` можно писать как `EX AF,AF` или `EXA`;
- `SLI` и `SLL` являются синонимами;
- поддерживаются половинки индексных регистров `XH`, `XL`, `YH`, `YL`, а также `HX`, `LX`, `HY`, `LY`.

## Примеры

Примеры находятся в `examples/`:

- `HELLO` - минимальная программа;
- `LOCAL` - локальные метки;
- `INCL` - `INCLUDE` из подкаталога;
- `MIXED` - несколько include-файлов из разных каталогов;
- `SAVE` - `SAVE`/`SAVEBIN`, `/N`, `BYTE`, `WORD`, `BLOCK`;
- `COND` - условная компиляция и пропуск неактивных веток;
- `DIAG` - диагностические `DISPLAY`, `WARNING` и `ASSERT`;
- `TASM` - TASM-совместимые `DISP`/`ENT`, `DD`/`DEFD`/`DWORD`, `DUP`/`EDUP`, `OUTPUT`/`OUTEND` и сгруппированные двоичные числа;
- `ERRORS` - намеренно ошибочный пример для проверки `/L` и активного `ERROR`.

В каждом каталоге примера есть `makefile` для Sprinter make и `build.bat` для запуска без make.

`ORGSELF.ASM` в исходниках предназначен для самосборки на целевой системе: он подключает `ORGASM.ASM` и сохраняет распакованное ядро как `CORE.BIN` с адреса `#4100`. Полный упакованный `ORGASM.EXE` собирается штатным `make` в репозитории.

## Дистрибутив

Zip-дистрибутив содержит `ORGASM.EXE`, `README`, `README.ENG`, `HISTORY`, `EXAMPLES/` и `DOCS/`. Текстовые файлы в дистрибутиве конвертируются в CP866. Образ дискеты содержит исполняемый файл, README/HISTORY и примеры; каталог `DOCS/` в образ дискеты не включается.
